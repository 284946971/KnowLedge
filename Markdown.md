# Markdown

## 简介

- 非常简洁、轻量的标记语言，就用语法就 10 种左右。
- 基于纯文本，可使用任何文本编辑器，文件名使用 .md 后缀。
- 原文高度可读。
- 通过工具渲染（render）成 HTML 后可在浏览器中活得更美观的展示效果。
- 广泛用于写文档、写博客、写书以及各种在线协作系统的文字交流等场景。

## Markdown 使用语法

``` text
- # 一级标题    段落前后要空行
- ## 二级标题
- *斜体* _斜体_（不要用_）
- **加粗**
- - 列表项 * 列表项（不要用*）
      - 列表子项
- ~~删除线~~
- > 引用      引用仅用于引用原文原句
- [百度](https://baidu.com)  链接
- ![图片](图片链接)   图片链接
- `var t = new Date();`  行内代码
- ```js
  var t = new Date();
  console.log(t);
  ```   代码块，并声明语言
- 列 1 | 列 2 | 列 3
  --- | ---: | ---
  1 | 2 | 3    表格
```

## Markdown 使用规范

### 语义

- 不要滥用强调格式。
- 不要滥用行内代码格式，只有代码才能使用行内代码格式，不要将其作为高亮强调格式。
- 不要滥用引用格式，不要将其作为高亮强调格式。

### 标题

- 有且只有一个一级标题作为文件标题。
- 标题级别要按序。
- 不要使用四级以上的标题。
- 前后要空行，但文件标题前面不用空。
- 不要在标题中使用强调、行内代码等格式。
- 整个标题可以是一个链接。

### 段落

前后要空行，尤其连续多个段落时，否则多数渲染器渲染出来会是一段。

### 强调

使用 `*` 而不是 `_`，因为 `_` 容易与变量名冲突。

### 链接

尽量用 `[标题](链接)` 的写法，避免直接用一长串裸链接，因为看起来会比较乱。链接标题尽量同其内容标题，上下文清楚时也可以用 `这里` 指代。

### 围栏代码块

- 前后要空行。
- 代码块的内容不需要缩进。
- 使用正确的语言标记，代码块内代码要合法。没有适用的语言标记时用 text，不能留空。

### 中文排版

#### 空格

- 中英文之间需要增加空格。
- 中文与数字之间需要增加空格。
- 数字与单位之间无需增加空格。
- 全角标点与其它字符之间不加空格。例外：裸链接后面需空格，不能紧跟标点，否则有些渲染器会把标点当成链接的一部分。
- 中文与链接、强调、行内代码等格式间是否加空格取决于内容，比如书写 markdown 的规范需要加，而书写中文的规范不需要加。

#### 全角与半角

- 主体是中文的时候（夹杂少了英文单词、术语、短句），应全部使用全角中文标点，不要随意混杂半角标点。
- 遇到完整的英文整句（引用）、特殊名词等时，其内容才可酌情使用半角标点。
- 数字使用半角字符。

#### 标点

- 中文列举间隔使用顿号（、）而不是逗号（，）。
- 列表项目结尾是在引出详细描述时，用冒号结束。
- 句末加句号，特别短（不是完整句子）可以例外，但是注意保持一致。

#### 其他

- 描述菜单、目录导航使用【xxx】>【xxx】的写法。
- 英语专有名词大小写、符号等遵循官方的写法，不要乱写，比如 macOS、Microsoft、GitLab 等。
- 英语标题除介词、连词等外应首字母大写。
